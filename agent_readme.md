# FoodFilter Agent

FoodFilter Agent 是一个围绕真实个人饮食数据库运行的本地点餐智能体。

它的目标不是回答“今天吃什么”，而是直接完成从偏好理解、店铺筛选、优惠领取、凑单比价到支付前确认的完整外卖决策链路。

`README.md` 用于说明 `foodfilter_skill` 的安装和使用；本文档记录 `foodfilter_agent` 的终局形态。

## Product Vision

FoodFilter Agent 的终局体验是：

```text
到饭点
  -> 主动读取用户食鉴数据库
  -> 理解白榜、黑榜、备注、群组来源和历史选择
  -> 打开外卖平台
  -> 搜索合适店铺
  -> 领取红包、店铺券、满减券
  -> 加购并凑满减
  -> 计算真实到手价
  -> 给出 5 个可确认方案
  -> 用户确认
  -> 停在支付前
```

用户不需要反复打开多个 App、不需要手动翻收藏夹、不需要自己算优惠券，也不需要把每家店都重新判断一遍。

用户只需要确认两件事：

- 今天选哪个方案
- 是否进入支付

## Core Difference

普通外卖推荐依赖平台排序、广告位、销量和评分。

FoodFilter Agent 依赖用户自己的食鉴数据库。

这些数据包括：

- 用户亲自记录的白榜店铺
- 用户明确拉黑的黑榜店铺
- 店铺备注里的推荐菜、避雷点和口味偏好
- 群组共享记录和来源用户
- 从小红书、点评、视频、图文链接中导入的店铺列表
- 用户每次接受、拒绝、改选后的行为记忆

这使 Agent 的推荐不是“平台认为你可能想吃什么”，而是“根据你已经维护过的真实偏好，帮你把这顿饭点好”。

## End-to-End Flow

```text
FoodFilter App / foodfilter_skill
  -> 用户饮食数据库
  -> Preference RAG
  -> Meal Planner
  -> Platform Operator
  -> Coupon And Bundle Planner
  -> Supervisor
  -> 5 个下单前方案
  -> 用户确认
```

### 1. Preference RAG

负责把食鉴数据变成可检索、可推理的偏好上下文。

输入：

- 食鉴 App 本地数据库
- 食鉴 App 导出的 JSON
- `foodfilter_skill` 生成的导入数据
- 群组共享店铺
- 用户历史点单行为

输出：

- 白榜优先级
- 黑榜排除规则
- 店铺备注摘要
- 推荐菜和避雷菜
- 群组偏好信号
- 预算、口味、距离和时间约束

### 2. Meal Planner

负责决定今天应该探索哪些店铺和菜品。

它需要同时考虑：

- 当前时间
- 用户预算
- 用户近期吃过什么
- 是否需要换口味
- 是否优先白榜
- 是否参考群组记录
- 是否需要跨平台比价
- 是否需要尝试新店

Planner 不直接点击手机，而是生成短步骤任务，让执行器逐步完成。

### 3. Platform Operator

负责操作真实手机上的外卖平台。

能力包括：

- 打开指定外卖 App
- 搜索店铺
- 进入店铺详情
- 识别菜品区、活动区、购物车和结算页
- 点击加购、规格、必选品、红包、优惠券和凑单入口
- 截图、OCR、结构化页面状态
- 在复杂页面中请求 LLM 判断下一步动作

目标是完成真实下单前操作，而不是只生成文本建议。

### 4. Coupon And Bundle Planner

负责把“看起来便宜”变成“真实便宜”。

它需要：

- 找到可用红包
- 找到店铺券和平台券
- 判断券是否适用于当前店铺
- 根据满减门槛反推合理加购区间
- 处理起送价、必选品、规格和配送费
- 在购物车中二次确认最终优惠

最终输出的每个方案都应包含：

- 店铺名
- 菜品列表
- 原价
- 优惠后价格
- 使用了哪些优惠
- 为什么推荐
- 来自用户白榜、群组还是新探索

### 5. Supervisor

负责监控 Agent 是否走在正确路径上。

它需要持续判断：

- 是否进入了广告页
- 是否误点活动浮层
- 是否进错店铺
- 是否卡在规格弹窗
- 是否缺少必选品
- 是否未达到起送价
- 是否已经到达支付前安全停点

当状态异常时，Supervisor 可以要求执行器重试、回退、关闭弹窗、重新搜索或停止任务。

### 6. Memory And Skill System

FoodFilter Agent 会把成功经验沉淀成平台专属 skill。

例如：

- 美团搜索结果页如何判断真实店铺卡片
- 哪些“抢”“神券”“活动”入口不应点击
- 店铺详情页如何进入菜品区
- 凑单页如何选择满足差额的商品
- 红包入口在哪里
- 哪些弹窗可以关闭

这些经验会随着执行更新，减少重复探索，提高下次点单效率。

## Technical Stack

FoodFilter Agent 可以组合以下技术实现：

- FoodFilter App 本地数据库
- `foodfilter_skill` 数据导入格式
- 本地 JSON / SQLite 偏好库
- RAG 检索与偏好排序
- Android ADB / iOS 自动化能力
- 截图与 OCR
- 多模态 LLM 页面理解
- 文本 LLM 任务规划
- Agent Supervisor 状态机
- AutoGLM / OpenClaw 类移动端执行框架
- 平台专属操作 skill
- 本地日志与行为记忆
- 小屏语音硬件交互

核心原则是：规则走主链路，LLM 处理复杂页面和未知状态，多模态模型作为视觉兜底。

## Interaction Model

FoodFilter Agent 更适合主动型交互。

示例：

```text
11:45
Agent: 午饭时间到了。我找到了 5 个方案。

1. 白榜川菜店，优惠后 28.6 元，含米饭和招牌小炒
2. 群组推荐日料，优惠后 35.2 元，配送更快
3. 新探索轻食，优惠后 24.9 元，符合最近低油偏好
4. 常点面馆，优惠后 19.8 元，最快送达
5. 白榜烧烤，优惠后 42.5 元，适合晚一点吃

用户: 选 1
Agent: 已加入购物车，已使用红包，停在支付前。
```

Agent 不替用户付款，不绕过平台确认，不隐藏最终价格。

## Hardware Vision

FoodFilter Agent 可以运行在两种形态上。

### Local Agent

适合开发者和高阶用户。

- 用户自己配置模型
- 连接自己的手机
- 使用本地电脑或本地服务器运行
- 适合调试和扩展平台 skill

### Dedicated Device

适合最终消费者。

- 小屏语音设备
- 到点主动提醒
- 展示 5 个方案
- 支持语音纠偏
- 与食鉴 App 同步数据
- 在手机外卖平台支付前停下

硬件不追求大而全，重点是把点外卖这一件事做到稳定、自然、少打扰。

## Relation To foodfilter_skill

`foodfilter_skill` 是数据入口。

它让外部智能体把图文、视频、链接、OCR 文本整理成食鉴 App 可导入格式。

`foodfilter_agent` 是执行系统。

它消费这些数据，并把用户偏好变成真实点餐动作。

```text
外部内容
  -> foodfilter_skill
  -> 食鉴 App
  -> FoodFilter Agent
  -> 外卖平台
  -> 支付前确认
```

## Commercial Direction

FoodFilter Agent 的长期商业价值来自三层：

- 食鉴 App：用户维护自己的饮食数据库
- Agent：把数据库变成自动点餐能力
- 硬件：把主动提醒、语音确认和本地执行封装成专用设备

软件层可以开放给开发者研究和个人使用。

商业化重点可以落在：

- 专用硬件
- 平台 skill 授权
- 高级模型能力
- 私有化本地 Agent 套件
