# FoodFilter Agent

FoodFilter Agent is a local food-ordering agent built around a user's real personal dining database.

Its goal is not to answer "What should I eat today?", but to complete the full food delivery decision chain: understanding preferences, filtering restaurants, collecting coupons, bundling items for discounts, comparing final prices, and stopping before payment confirmation.

`README.md` explains how to install and use `foodfilter_skill`; this document describes the intended usage model of `foodfilter_agent`.

## Product Vision

The FoodFilter Agent experience is:

```text
Meal time arrives
  -> Proactively read the user's FoodFilter database
  -> Understand whitelist, blacklist, notes, group sources, and past choices
  -> Open a food delivery platform
  -> Search for suitable restaurants
  -> Claim red packets, restaurant coupons, and platform discounts
  -> Add items and bundle for threshold discounts
  -> Calculate the real final price
  -> Present 5 confirmable options
  -> User confirms
  -> Stop before payment
```

Users should not need to repeatedly open multiple apps, browse saved lists manually, calculate coupons by themselves, or re-evaluate every restaurant from scratch.

The user only needs to confirm two things:

- Which option to choose today
- Whether to proceed to payment

## Core Difference

Typical food delivery recommendations depend on platform rankings, ad placements, sales volume, and ratings.

FoodFilter Agent depends on the user's own FoodFilter database.

This data includes:

- Restaurants personally whitelisted by the user
- Restaurants explicitly blacklisted by the user
- Recommended dishes, warning notes, and taste preferences in restaurant notes
- Group-shared records and source users
- Restaurant lists imported from Xiaohongshu, review sites, videos, image-text posts, and links
- Behavioral memory from every accept, reject, and change decision

This makes the Agent's recommendations not "what the platform thinks you might want", but "what should be ordered based on the real preferences you have already maintained".

## End-to-End Flow

```text
FoodFilter App / foodfilter_skill
  -> User dining database
  -> Preference RAG
  -> Meal Planner
  -> Platform Operator
  -> Coupon And Bundle Planner
  -> Supervisor
  -> 5 pre-order options
  -> User confirmation
```

### 1. Preference RAG

Turns FoodFilter data into searchable and reasoned preference context.

Inputs:

- FoodFilter App local database
- JSON exported from FoodFilter App
- Imported data generated by `foodfilter_skill`
- Group-shared restaurants
- User's historical ordering behavior

Outputs:

- Whitelist priorities
- Blacklist exclusion rules
- Restaurant note summaries
- Recommended dishes and dishes to avoid
- Group preference signals
- Budget, taste, distance, and time constraints

### 2. Meal Planner

Decides which restaurants and dishes should be explored for the current meal.

It considers:

- Current time
- User budget
- What the user has eaten recently
- Whether the user should change taste direction
- Whether whitelisted restaurants should be prioritized
- Whether group records should be referenced
- Whether cross-platform price comparison is needed
- Whether new restaurants should be explored

The Planner does not directly operate the phone. It generates short tasks and lets the executor complete them step by step.

### 3. Platform Operator

Operates real food delivery platforms on a real phone.

Capabilities include:

- Open the target food delivery app
- Search for restaurants
- Enter restaurant detail pages
- Recognize dish areas, activity areas, carts, and checkout pages
- Tap add-to-cart, specifications, required items, red packets, coupons, and bundle-entry points
- Capture screenshots, run OCR, and structure page state
- Ask an LLM to decide the next action on complex pages

The goal is to complete real pre-payment ordering actions, not merely generate text suggestions.

### 4. Coupon And Bundle Planner

Turns "looks cheap" into "actually cheap".

It needs to:

- Find available red packets
- Find restaurant coupons and platform coupons
- Determine whether a coupon applies to the current restaurant
- Infer reasonable add-on price ranges from discount thresholds
- Handle minimum order price, required items, specifications, and delivery fees
- Reconfirm the final discount inside the cart

Each final option should include:

- Restaurant name
- Dish list
- Original price
- Discounted final price
- Discounts used
- Reason for recommendation
- Whether it comes from the user's whitelist, a group record, or new exploration

### 5. Supervisor

Monitors whether the Agent is on the correct path.

It continuously checks:

- Whether the Agent entered an ad page
- Whether it tapped a floating campaign layer by mistake
- Whether it entered the wrong restaurant
- Whether it is stuck in a specification popup
- Whether required items are missing
- Whether the minimum order price has not been reached
- Whether it has reached the safe stop point before payment

When the state is abnormal, the Supervisor can ask the executor to retry, go back, close a popup, search again, or stop the task.

### 6. Memory And Skill System

FoodFilter Agent turns successful experience into platform-specific skills.

Examples:

- How to identify real restaurant cards on Meituan search result pages
- Which "flash deal", "special coupon", or "campaign" entries should not be tapped
- How to reach the dish area from a restaurant detail page
- How to select items that satisfy the remaining bundle threshold
- Where red packet entries are located
- Which popups can be closed

These experiences are updated during execution, reducing repeated exploration and improving the next ordering run.

## Technical Stack

FoodFilter Agent can be built by combining:

- FoodFilter App local database
- `foodfilter_skill` import format
- Local JSON / SQLite preference store
- RAG retrieval and preference ranking
- Android ADB / iOS automation
- Screenshot and OCR
- Multimodal LLM page understanding
- Text LLM task planning
- Agent Supervisor state machine
- AutoGLM / OpenClaw-style mobile execution framework
- Platform-specific operation skills
- Local logs and behavior memory
- Small-screen voice hardware interaction

The core principle: rules handle the main path, LLM handles complex pages and unknown states, and multimodal models act as visual fallback.

## Interaction Model

FoodFilter Agent is designed for proactive interaction.

Example:

```text
11:45
Agent: Lunch time. I found 5 options.

1. Whitelisted Sichuan restaurant, 28.6 after discount, includes rice and signature stir-fry
2. Group-recommended Japanese restaurant, 35.2 after discount, faster delivery
3. New light-meal option, 24.9 after discount, matches recent low-oil preference
4. Frequently ordered noodle shop, 19.8 after discount, fastest arrival
5. Whitelisted barbecue restaurant, 42.5 after discount, better for later

User: Choose 1
Agent: Added to cart, red packet applied, stopped before payment.
```

The Agent does not pay on behalf of the user, bypass platform confirmation, or hide the final price.

## Hardware Vision

FoodFilter Agent can run in two forms.

### Local Agent

For developers and advanced users.

- Users configure their own model
- Connect their own phone
- Run on a local computer or local server
- Suitable for debugging and extending platform skills

### Dedicated Device

For end consumers.

- Small-screen voice device
- Proactive reminders at meal time
- Display 5 options
- Support voice correction
- Sync with FoodFilter App data
- Stop before payment on the mobile food delivery platform

The hardware does not aim to do everything. The focus is making food delivery ordering stable, natural, and low-interruption.

## Relation To foodfilter_skill

`foodfilter_skill` is the data entry tool.

It lets external agents turn image-text posts, videos, links, and OCR text into a format that can be imported into FoodFilter App.

`foodfilter_agent` is the execution system.

It consumes that data and turns user preferences into real ordering actions.

```text
External content
  -> foodfilter_skill
  -> FoodFilter App
  -> FoodFilter Agent
  -> Food delivery platform
  -> Pre-payment confirmation
```
